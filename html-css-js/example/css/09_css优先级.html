09. css优先级

(1) 相同样式优先级

当设置相同样式时，后面的优先级较高，也就是跟引入 的先后顺序有关，但是一般不建议出现重复设置相同样式的情况

(2) 内部样式与外部样式

内部样式与外部样式优先级相同，如果都设置了相同样式，那么后写的引入方式优先级高，也就是跟样式表引入的顺序有关

(3) 单一样式优先级

style行间 >id >class >tag >* >继承

注：如果按照权重的方式进行比较，可以认为大致如下

  • style行间 权重：1000
  • id 权重：100
  • class 权重：10
  • tag 权重：1

(4) !important

提升样式优先级，这不是规范方式，不建议使用（此外，该方式不能针对继承的属性进行优先级的提升）

(5) (标签+类)vs单类

正常情况下，标签和类的优先级会比单类的优先级大

(6) 群组优先级

群组选择器与单一选择器的优先级相同，因而，如果同时存在相同的标签，那么靠后写的优先级高

(7) 层次优先级

对于层次选择器，一般建议层次在三层以内，多了会变得复杂，代码可读性变差

  1. 权重比较
    • ul li .box p input{}   -> 1+1+10+1+1
    • .hello span #elem{}   -> 10+1+100
  2. 约分比较
    • ul li .box p input{}   -> li p input
    • .hello span #elem   -> #elem
